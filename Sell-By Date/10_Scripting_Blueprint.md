## 1. The fixed spine (do not relitigate these)
- **Premise:** a failing modern ad agency can sell modern products to the past; each episode = one product → one era → one sale, with consequences.
- **Format:** 90-second **vertical**, self-contained, episodic. **No pilot** (short-form needs none; every episode onboards a cold viewer in ~3s).
- **Controlling idea:** *Anything can be sold to anyone — but every sale rewrites the world, because ambition always outruns conscience.*
- **Tone ("X meets Y"):** *Mad Men meets Rick and Morty, staged in a Blackadder court.*
- **Core value swing:** existence ↔ erasure (control ↔ chaos underneath).
- **Locked decisions:** origin = *accidental discovery*; antagonist = *self-erasure through overreach* ("history's immune system"); cast = *three-hander, Drew the clear solo lead*.
- **Output format:** the **spec-screenplay** format (header block + camera-blind), NOT a Hollywood feature script.

## 2. The characters (engines + voices — full detail in 02 & 07)
- **DREW** (lead) — misplaced commercial priority; meets catastrophe, sees pipeline. Voice: **Don Draper**.
- **KAI** — reckless-builder pride; treats disaster as a successful launch. Voice: **Russ Hanneman**.
- **MARGO** — the accurate genius nobody listens to; delivers the stakes via the **"English, Doc!"** device. Voice: **Ian Malcolm / Doc Brown**.
- **Period guests** (kings, generals) — grand, vain, huge reactions, elaborate wit. Voice: **Blackadder**.

## 4. The episode template (repeatable ~90s beat sheet)
1. **HOOK (0–3s):** a first-frame-compliant image of the clash (modern person/product dropped into the past) — body in motion / extreme face / visible incongruity / reaction-to-impact. Lay the **first-person Impossible-Result caption** with a $ figure (per `08`).
2. **THE SETUP:** the era, the buyer, the product, and the cultural obstacle (why selling it is hard — the clash).
3. **THE SELL / ESCALATION:** the product hits the era; the **period guest's BIG reaction** carries the comedy; beats build by *but / therefore*.
4. **THE TURN:** it works too well / backfires / a consequence (death, ripple, history visibly changes).
5. **THE BUTTON:** back with the crew — the **margin/greed payoff** (fragmented exchange, concrete numbers), **Margo's stakes warning** ("English, Doc!"), Drew's **greed-over-stakes** line.
6. **CLIFFHANGER END CARD:** `SELL-BY DATE — Next: how do you sell [PRODUCT] to [ABSURD BUYER]?`
> *Episode 1 ("Cold Call") is the **origin variant** of this template: the first sale is accidental, and the button is the realization that launches the series.*

## 5. Dialogue — the five laws (full spec in `07`)
1. Picturable/concrete (ban "we disappear," "that's a market").
2. Literal, never a riddle (no pun-as-logic).
3. No line assumes a premise the audience doesn't have yet.
4. Smart exposition via **"English, Doc!"** (real jargon → "English" → specific plain line); realizations are **fragmented exchanges + concrete margin**, never a tidy monologue.
5. Comedy from **character + situation + spectacle** (big period reactions), not dry quips. Read every line aloud.

## 6. On-screen text — the rules (full spec in `08`)
≤14 words, lands in 3s, ≥2 triggers, **one concrete number** ($20K move), completes-not-labels the visual, 3 variants for A/B. House hook formula = **First-Person Impossible Result**; end card = **Cliffhanger Question**.

## 8. How to work (process rules that saved iterations)
- **One version + one narrow binary question.** Never dump 3–5 option menus.
- **Ground dialogue/text in real references — search, don't improvise voice.** When a line feels off, pull the actual source cadence before rewriting.
- **Match the established comedy register** (big period fish-out-of-water + character-driven). Do NOT drift into dry office/corporate deadpan.
- **Logic discipline:** characters only know what they witnessed; physical/spatial state must track; reactions fit the actual stimulus.
- **Never declare "final"/save-as-locked without explicit user sign-off.**
- **Don't force conventions that don't fit the medium.**
- **Drive delivery:** generate a local `.docx` (screenplay-formatted via the docx skill) and tell the user to drag-drop. **Never upload to Drive yourself.**
